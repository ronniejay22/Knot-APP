# Security Policy

Knot is a personal relationship app that stores a sensitive, private vault of
user data. We take reports of security issues seriously and appreciate
responsible disclosure.

## Supported Versions

Knot is a single, continuously-developed application rather than a set of
versioned releases. Only the latest code on the `main` branch receives security
fixes. There is no support for older snapshots or forks.

## Reporting a Vulnerability

**Please do not open a public issue or pull request for security problems.**
Public disclosure before a fix is available puts users' data at risk.

Instead, report it privately through GitHub's private vulnerability reporting:

1. Go to the repository's **Security** tab.
2. Select **Advisories → Report a vulnerability**.
3. Fill in the details — what you found, how to reproduce it, and the potential
   impact.

This opens a private channel visible only to the maintainer.

### What to expect

This is an early-stage, personally-maintained project, so responses are
best-effort rather than bound by a formal SLA:

- **Acknowledgement:** we aim to confirm we've received your report within a few
  days.
- **If accepted:** we'll work on a fix and coordinate privately with you on
  timing before any public disclosure, and credit you if you'd like.
- **If declined:** we'll explain why we don't consider it a vulnerability (for
  example, it's out of scope or working as intended).

Because Knot handles a sensitive encrypted vault, giving us a reasonable chance
to address an issue before disclosing it publicly is genuinely appreciated.
